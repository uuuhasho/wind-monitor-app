# 🚢 船期日曆與奶昔系統 - 碼頭(W12)擴充修改指南

此文件紀錄了在不破壞現有正規表達式（Regex）與風速判定邏輯的前提下，新增「碼頭（Terminal）」標示的修改重點。

## 1. 船期日曆系統 (`船期日曆.js` 與其 HTML)
**目標：** 在日曆上加入 `@W12` 標籤，並在 API 輸出時乾淨地將其切分出來。

*   **前端輸入 (Index.html)：** 
    *   新增一個碼頭輸入框，或在送出資料時檢查是否有輸入碼頭。
*   **寫入日曆 (`updateCalendarFromList`)：**
    *   加入防呆機制：轉大寫比對，只認 `W12`。
    *   若是 W12，則在日曆標題 `fullTitle` 的**最前面**加上 `@W12 ` 前綴。
    *   *格式範例：`@W12 Cobia(160K)-Qatar`*
*   **API 輸出 (`handleApiRequest`)：**
    *   **剝洋蔥前處理：** 在進行任何符號切割前，先檢查 `shipNameOnly` 是否以 `@` 開頭。
    *   如果是，抓出 `@` 到第一個空白之間的文字當作 `terminal`（即 `W12`），然後把 `@W12 ` 這段從字串中**刪除**。
    *   剩下的字串完美對接舊有邏輯（找 `-` 與 `()` 切割）。
    *   在 API 回傳的 JSON 陣列中新增 `t: terminal` 屬性。

## 2. 奶昔系統後端 (`gs_NV4.7.5.js`)
**目標：** 接收 API 傳來的碼頭資訊，並將其寫入 Firebase 供前端讀取。

*   **API 讀取與船期表 (`getScheduleList`)：**
    *   在處理 `apiData` 的 `map` 迴圈中，讀取 `item.t`。
    *   將碼頭資訊放入回傳物件中：`{ d: item.d, ship_name: shipName, limit_speed: limitSpeed, terminal: item.t }`。
*   **啟動監控 (`startMonitoring`)：**
    *   函式新增 `terminal` 參數。
    *   寫入 `daily_records/.../config` 時，增加 `"terminal": terminal || ""`。
    *   寫入 `active_status` 時，增加 `"terminal": terminal || ""`。
*   **自動抓取 (`checkDailySchedule`)：**
    *   呼叫 API 找到今日船隻時，把 `todayShip.t` 接接起來，傳給 `startMonitoring`。
*   **手動觸發 (`doGet` - `set_mode`)：**
    *   從前端接收 `e.parameter.terminal`，一併傳給 `startMonitoring`。

**目標：** 增加手動輸入介面，並在看板與船期表上顯示 W12 標籤。

*   **手動監控視窗 (`manualModal`)：**
    *   新增一個「碼頭 (非必填)」的 `<input>`，讓它變成選單，但只有`W12`可以選。
    *   修改 `setMode('start')`，將碼頭數值讀取出來並傳送給 GAS API。
*   **即時動態看板 (`updateDynamicBox`)：**
*   **未來船期表 (`renderScheduleTable`)：**

    *   如果是 W12，可以在「船名」欄位直接組合顯示，如 `Cobia <span style="color:orange;"> @W12</span>`，以節省表格空間。
